Doomsday Unoffical Debian Packages
G'day.

I've built unoffical debian packages of Doomsday and a variety of Jpacks. I would also like to make my packages available for other people to use, comment and improve on, but before I can do that I need to sort out some redistribution issues.

As this is the largest gathering of Doomsday users and developers I thought this was an appropriate place to post.

The packages I have created are:

deng-1.8.6 (OK to distribute - 9 Mar 2005)
This is the Doomsday Engine version 1.8.6

deng-jdoom-awp-3.03

This is Abb's Alternate Weapon Pack 3.03, It was made into a pk3 file instead of a Windows exe file. I understand that Abb no longer is involved with this. Are there any problems with me distributing this as a debian package ?

deng-jdoom-bamteleports-20030629 (OK to distribute - 8 Mar 2005)
This is a script found here http://forums.newdoom.com/showthread.php?t=16068

deng-jdoom-ben2k9lights-20050211 (OK to distribute - 9 Mar 2005)
This is a script found here http://forums.newdoom.com/showthread.php?t=26585

deng-jdoom-ep-2.03 (OK to distribute - 8 Mar 2005)
This is the jDoom environment pack 2.03 from Slide's website. It contains both the core and final doom extension merged into 1 pk3 file.

deng-jdoom-lavalights-20040830
This is a script found here http://forums.jfiles.org/viewtopic.php?t=35
This script cause doom1 to die on E3M6 so I modified it to not load on doom1.
KuriKai - Are there any problems with me distributing this as a debian package ?

deng-jdoom-rp-1.01 (OK to distribute - 7 Mar 2005)

deng-jdoom-tp-20050306 (OK to distribute - 9 Mar 2005)
This is the JDTP pack from jfiles.org, it contains jdtp-zip-full-base.zip and jdtp-textures-doom1.pk3.zip.

deng-jdoom-ui-1.07 (OK to distribute - 8 Mar 2005)
This is the 1.07 release of the jDoom UI pack from Slide's website.
